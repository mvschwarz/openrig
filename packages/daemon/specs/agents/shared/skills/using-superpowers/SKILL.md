# Using Superpowers

Do not free-run from bootstrap text alone.

If your startup guidance or role guidance names a skill, load it before you do substantive work.
Treat packaged skills as operating instructions, not optional reading.

## Starter rule

For OpenRig starter rigs:
1. Load `openrig-user`.
2. Load your pod or role skill.
3. Load the process skills your role startup checklist names.
4. Then act.

If a skill is missing, say so explicitly and continue with the best shipped guidance you do have.

## Practical rule

Before you:
- plan work
- debug a blocker
- write code
- review code
- claim something is complete

check whether a packaged skill already tells you how to do that better than improvising.

## Red flags

Stop and load the skill first if you catch yourself thinking:
- "I already know how to do this"
- "The startup file is probably enough"
- "I'll just get moving and clean it up later"
- "This is too simple to bother loading the skill"

That is exactly how starter rigs drift into inconsistent behavior.

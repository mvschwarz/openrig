# OpenRig v0.3.1

## Summary

OpenRig v0.3.1 builds on v0.3.0 with operator-facing UX work, the headline
Claude auto-compaction policy, a dashboard + For You visual refresh, the
Plugin Primitive v0 surface (`rig plugin`), Library Explorer finishing
(skills + plugins + specs with real folder browsing), Settings Destination
Explorer, action outcome + inline error UX, and a comprehensive privacy
hygiene pass.

No schema-breaking migrations. Existing v0.3.0 databases upgrade on first
`rig daemon start` (adds migration `040_workflow_specs_diagnostic` — two
new diagnostic columns on `workflow_specs` cache, `ALTER TABLE ADD COLUMN`
with safe defaults).

## Included In This Release

- **Claude Auto-Compaction Policy** — operator-configurable session
  auto-`/compact` with safe defaults (opt-in default-off; no behavior
  change unless explicitly enabled). 7 new ConfigStore keys; 6-stage
  daemon-to-LLM lifecycle (pre-prep prompt → compact → handshake →
  restore → compliance audit → cooldown); PreCompact hook +
  SessionStart/UserPromptSubmit bridge.
- **Plugin Primitive v0** — `rig plugin` command surface (read-only at
  v0: `list`, `show`, `used-by`, `validate`). Plugin discovery from
  `$OPENRIG_HOME/plugins/`, vendored-and-provenance pattern. No
  `install` verb at v0; deferred to 0.3.2.
- **Library Explorer Finishing** — skills + plugins + specs as a
  fully operator-facing surface with real folder browsing. New
  daemon-owned library discovery API (`/api/skills/library`,
  `/api/skills/:id/files/...`, `/api/plugins/:id/files/...`).
- **Settings Destination Explorer** — settings became a 4-item
  Explorer destination matching Topology / Project / Library / For-You
  pattern, with `/settings`, `/settings/policies`, `/settings/log`,
  `/settings/status` sub-routes.
- **Dashboard And For You Visual Refresh** — both routes adopt a
  shared "vellum" surface language (translucent stone-tinted cards,
  paper-grid background, ambient shadow, mono+dot kind indicators,
  L-shaped corner-bracket registration marks). Single source of truth
  between production and `/lab/vellum-lab`.
- **For You Storytelling Adapter** — Progress / Shipped / Incident /
  Approval card kinds wired to real data (missions, slices, activity
  feed, queue items). ApprovalCard surfaces real queue items waiting
  on approval.
- **Action Outcome + Inline Error Surface** — queue-item verb actions
  (Approve/Deny/Route/Hold/Drop/Annotate/Handoff) render outcomes
  immediately on success and surface failures inline without
  silently reverting. Closes a demo-blocking silent-revert bug.
- **Mobile Drawer Behavior** — 375px Explorer drawer now layers above
  the rail tray for Settings / Project / Library / For-You. Topology
  mobile drawer is intentionally hidden in 0.3.1; scheduled for full
  restoration in 0.3.2.
- **Vendored Skill Provenance** — vendored skills declare upstream
  lineage via `metadata.openrig` frontmatter and (when modified) a
  companion `OPENRIG.md` sidecar. Applied to all 10 process-skill
  surfaces.
- **Workflow specs diagnostic surface** — migration `040` adds
  `status` + `error_message` columns to the `workflow_specs` cache so
  the Library UI can render parse/validation errors as diagnostic
  rows instead of hiding them.
- **Privacy hygiene pass** — three filter-repo passes scrubbed
  personal data, internal references, and historic versions from
  history; commit subjects and bodies polished for public surface.
  Backup repos preserved for rollback.

## User-Facing Changes

### Claude Auto-Compaction Policy

Operator-configurable auto-`/compact` for Claude sessions, with safe
defaults:

- **Opt-in default-off**: nothing changes unless you explicitly enable
  the policy. With no policy configured, the daemon never sends an
  auto-`/compact`.
- **7 new ConfigStore keys** in the `policies.claude_compaction.*`
  namespace: `enabled`, `threshold_percent` (strict integer 1-100),
  `compact_instruction` (operator-supplied; appended to `/compact`
  args when set), `message_inline`, `message_file_path`,
  `pre_compact_instruction`, `post_restore_audit_instruction`. Strict
  validation across all source layers (`set`/`POST`/env/file). Invalid
  env values fall back to defaults with a stderr warning rather than
  silently coercing.
- **5 operator-editable prompt surfaces** in Settings → Policies UI:
  pre-compaction prep, compact instruction, post-compaction restore
  (inline), restore file path, post-restore audit. Daemon-owned
  wrappers (usage/threshold framing, trust-channel preservation,
  marker paths, read-depth enforcement, turn-boundary handshake,
  dedup + cooldown) are non-editable.
- **6-stage lifecycle**: pre-compact prep prompt (save-game-before-quit
  mental-model restore map) → `/compact` with operator instructions
  → post-compact turn-boundary handshake → restore prompt with
  defense-in-depth fallback chain → compliance audit (forces
  FULL/PARTIAL/NOT_READ table) → 10-minute cooldown.
- **Safety hardening**: typed prompt drafts classified as attention
  state so auto-`/compact` retries later instead of overwriting human
  input. Send-failure does not advance dedup state. Re-arm requires
  threshold-crossing.

### New `rig plugin` Command (read-only at v0)

```
rig plugin list                   # Discoverable plugins across vendored + runtime caches
rig plugin show <id>              # Plugin manifest + skills + hooks + mcp servers
rig plugin used-by <id>           # Agents referencing this plugin
rig plugin validate <path>        # Validate plugin manifest + skill frontmatter
```

No `install` verb at v0 — installation is explicit operator copy or
symlink to `$OPENRIG_HOME/plugins/<plugin-id>/`. See each plugin's
`OPENRIG-INSTALL.md` for the documented copy/symlink workflow. A
`rig plugin install <substrate-path>` verb is deferred to 0.3.2.

### Library Explorer Finishing

The Library destination (skills + plugins + specs) is now fully
operator-facing:

- No duplicate top-level entries (`> SKILLS` / `> PLUGINS` rows removed
  from `SpecsTreeView`; bottom-row clicks navigate + expand the tree).
- Plugins above Skills in the tree.
- 32 OpenRig-managed shared skills are now discoverable in production
  VM environments (previously empty due to workspace-relative path
  lookup).
- Daemon-owned library discovery API: `GET /api/skills/library`,
  `GET /api/skills/:id/files/list?path=...` + `/files/read?path=...`,
  parallel routes under `/api/plugins/:id/`.
- Real file-browser + docs-browser on plugin and skill detail pages.
  Markdown auto-renders; non-markdown files render as text. Folder
  navigation works.
- Rolled-up index pages at `/specs/skills` and `/specs/plugins` with
  flat-row listings, source labels, file counts, and entry links.

### Settings Destination Explorer

Settings is now a top-level Explorer destination matching the Topology /
Project / Library / For-You pattern:

- `/settings` (general config keys form)
- `/settings/policies` (Claude auto-compaction policy form lives here)
- `/settings/log`
- `/settings/status`

Old top-row tab navigation removed. Shared `SettingsPageShell` chrome
across all 4 sub-routes.

### Dashboard And For You Visual Refresh

A coordinated visual refresh to a shared "vellum" surface language:

- **Dashboard `/`** — full rewrite over new `dashboard/vellum/`
  primitives. Hero typography ("WELCOME BACK") at display-lg + headline-
  bold. Tactical instrument-panel stats line with tabular numerals.
  Six destination cards (Topology / Project / For You / Library /
  Search / Settings) share the numeral-layout treatment. Real-data
  hooks: `useRigSummary`, `usePsEntries`, `useSpecLibrary`.
- **For You `/for-you`** — both card systems unified to the vellum
  recipe: storytelling band (`CardShell` rewritten with bg-stone-100/45
  + backdrop-blur + ambient shadow + corner brackets) and queue-item
  `FeedCard.tsx` (same outer chrome; vellum bordered-no-fill action
  buttons with hover-invert and 44px touch targets).
- **Single source of truth** between `/dashboard` and
  `/lab/vellum-lab` via `packages/ui/src/components/dashboard/vellum/`.
- **Lab routes** — `/lab/card-previews`, `/lab/vellum-lab`,
  `/lab/vellum-bg/{a-large,b-small,c-allover}` are checked-in
  experiment surfaces for designer iteration. Reachable in production
  by direct URL; not linked from main navigation.

### For You Storytelling Adapter

The storytelling band at top of `/for-you` now wires four card kinds to
real data:

- **Progress** — first 2 active missions (`useMissionDiscovery`)
- **Shipped** — slices with status = shipped/complete/done (capped at 3;
  from `useSlices`)
- **Incident** — slices with status = blocked/failed/danger (capped at 3)
- **Approval** — queue items waiting on approval, from
  `useActivityFeed` + `classifyFeed` (capped at 2)

`ConceptCard` component is preserved in source but not emitted by the
production adapter; a deliberate data-source decision is scheduled for
0.3.2.

### Action Outcome And Inline Error Surface

Queue-item verb actions (Approve/Deny/Route/Hold/Drop/Annotate/Handoff):

- On success, `ActionOutcomePanel` renders immediately ("Approved by X",
  "Routed by X to Y") — no waiting for audit-log roundtrip.
- On error, a tertiary-bordered error block renders below the verb
  buttons with the daemon's error message; verb-selection state is
  preserved so the operator can correct and retry.
- Replaces the prior silent-revert UX. Regression test guards the class
  explicitly.

## Operator / Setup Notes

After upgrading the published CLI:

```bash
npm install -g @openrig/cli@0.3.1
rig --version    # expect 0.3.1
rig daemon stop
rig daemon start
```

Migration `040_workflow_specs_diagnostic` applies automatically on first
post-upgrade daemon start. It is a non-destructive `ALTER TABLE ADD
COLUMN` with safe defaults (existing rows retroactively get
`status: 'valid'`). No manual action required.

For Claude auto-compaction policy: opt-in default-off. Visit
`/settings/policies` to configure. Recommended for sessions where
context exhaustion is a recurring operational concern. Read the docs
section in the Settings UI before enabling.

For the new `rig plugin` surface: no behavior change unless you install
plugins. Bundled `openrig-core` plugin ships with the daemon (11 skills).
Operators can install `gstack` (45 skills) or `obra-superpowers`
(14 skills) by following each plugin's `OPENRIG-INSTALL.md` workflow.

## Known Limitations

- **`rig down <name>` still returns HTTP 404** — name-based teardown
  still doesn't resolve through the daemon down route at v0.3.1. Use
  `rig down <rigId> --delete` instead. Carries forward as known issue
  to v0.3.2.
- **`rig queue list --json --limit` and `rig view list/show --json`
  flag inconsistency** — flags scaffolded but not fully wired. Use
  human-readable output for now. Carries forward to v0.3.2.
- **Topology mobile drawer hidden** at 375px viewports in 0.3.1
  (intentional; clicking the hamburger triggered a pre-existing
  `TopologyTableView` renderer cascade). Full restoration scheduled
  for 0.3.2 via a dedicated render-path slice.
- **`ConceptCard` component preserved in source** but not emitted by
  the production For You adapter. A deliberate data-source decision is
  scheduled for 0.3.2.
- **`/lab/*` routes ship to production builds** intentionally per
  founder direction; 0.3.2 adds an index page + per-lab banner +
  manifest pattern.
- **Plugin install ergonomics** — v0 plugin install is operator-driven
  copy or symlink; first-class `rig plugin install <substrate-path>`
  is deferred to 0.3.2.
- **Lifecycle classifications** remain truthful but still rely on
  heuristics rather than perfect provider-native primitives in some
  recovery paths.
